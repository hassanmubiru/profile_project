WHAT IS A VARIABLE?
  A variable is a named container that stores data in memory.
  
  Syntax: variable_name = value
  
  Example:
    age = 25              # Variable named 'age' storing number 25
    name = "Alice"        # Variable named 'name' storing text "Alice"
    topics = [..., ..., ...]  # Variable storing a list

---

VARIABLE EXAMPLE 1: User Object Variable
Location: profile_app/views.py (line 30-31)

Code:
  def profile_view(request):
      # VARIABLE EXAMPLE 1: Get current logged-in user
      current_user = request.user    # ← This is a variable

What's happening:
  • 'current_user' is the variable name
  • 'request.user' is what we're storing (the User object)
  • The variable now holds a reference to the logged-in user
  • We can use current_user throughout the function

Usage example:
  current_user.username       # Get username from the variable
  current_user.email          # Get email from the variable
  current_user.first_name     # Get first name from the variable

VARIABLES IN MODELS (Database)
Location: profile_app/models.py (lines 15-30)

Code:
  class Profile(models.Model):
      # Variables defining database fields
      user = models.OneToOneField(User, on_delete=models.CASCADE)
      age = models.IntegerField(default=0)
      bio = models.TextField(default="", blank=True)
      topics = JSONField(default=list, blank=True)
      created_at = models.DateTimeField(auto_now_add=True)
      updated_at = models.DateTimeField(auto_now=True)

What's happening:
  • Each line defines a field variable in the database
  • 'user': Stores a reference to a User (one-to-one relationship)
  • 'age': Stores an integer number
  • 'bio': Stores text
  • 'topics': Stores a JSON list (could be: ["variables", "loops"])
  • 'created_at': Stores when the profile was created
  • 'updated_at': Stores last update time

In Python code:

  my_profile.bio = "Learning"      # Setting another variable
  my_profile.topics = ["var", "loop", "func"]  # Setting list variable

2️⃣ LOOPS - COMPLETE WALKTHROUGH

WHAT IS A LOOP?
  A loop repeats a block of code for each item in a collection.
  
  Pattern:
    for item in collection:
        # Do something with item
        # This block runs once per item

---

LOOP EXAMPLE 1: Python For Loop with enumerate()
Location: profile_app/views.py (lines 57-68)

Code:
  # LOOP EXAMPLE 1: Process topics with enumerate()
  processed_topics = []
  
  for index, topic in enumerate(today_topics, start=1):
      formatted_topic = f"Lesson {index}: {topic.upper()}"
      processed_topics.append(formatted_topic)

How it works (step by step):
  
  today_topics = ["variables", "loops", "functions"]
  
  Iteration 1:
    index = 1, topic = "variables"
    formatted_topic = "Lesson 1: VARIABLES"
    processed_topics = ["Lesson 1: VARIABLES"]
  
  Iteration 2:
    index = 2, topic = "loops"
    formatted_topic = "Lesson 2: LOOPS"  
    processed_topics = ["Lesson 1: VARIABLES", "Lesson 2: LOOPS"]
  
  Iteration 3:
    index = 3, topic = "functions"
    formatted_topic = "Lesson 3: FUNCTIONS"
    processed_topics = [
        "Lesson 1: VARIABLES",
        "Lesson 2: LOOPS",
        "Lesson 3: FUNCTIONS"
    ]

Key concepts:
  • enumerate(today_topics, start=1) provides:
    - index: Position (1, 2, 3 because we said start=1)
    - topic: Current item from the list
  • .upper() converts text to uppercase
  • f-string: f"..." allows inserting variables with {variable}
  • .append() adds item to the list

---

LOOP EXAMPLE 2: Django Template Loop
Location: profile_app/templates/profile.html (lines 127-145)

Code:
  <ul class="topics-list">
      {% for topic in today_topics %}
          <li class="topic-item">
              <span class="topic-number">{{ forloop.counter }}</span>
              <span class="topic-name">{{ topic|upper }}</span>
          </li>
      {% endfor %}
  </ul>

How it works (step by step):

  today_topics = ["variables", "loops", "functions"]
  
  Iteration 1:
    forloop.counter = 1
    topic = "variables"
    Output: <li><span class="topic-number">1</span><span...>VARIABLES</span></li>
  
  Iteration 2:
    forloop.counter = 2
    topic = "loops"
    Output: <li><span class="topic-number">2</span><span...>LOOPS</span></li>
  
  Iteration 3:
    forloop.counter = 3
    topic = "functions"
    Output: <li><span class="topic-number">3</span><span...>FUNCTIONS</span></li>

Key concepts:
  • {% for ... in ... %} starts the loop (Django template syntax)
  • {{ topic }} displays the current item
  • {{ forloop.counter }} is the current iteration number (1, 2, 3)
  • |upper is a filter that converts to uppercase
  • {% endfor %} closes the loop

Available forloop variables:
  {{ forloop.counter }}     # 1, 2, 3... (current position)
  {{ forloop.counter0 }}    # 0, 1, 2... (starts from 0)
  {{ forloop.first }}       # True only on first iteration
  {{ forloop.last }}        # True only on last iteration
  {{ forloop.length }}      # 3 (total items in list)

LOOP SUMMARY TABLE
═════════════════════════════════════════════════════════════════════════════

Syntax          | Purpose              | Output                | Example
────────────────┼──────────────────────┼──────────────────────┼─────────────────
for i in list   | Simple iteration     | Accesses each item   | for x in [1,2,3]
enumerate()     | Index + item         | index, item values   | for i, x in enum()
range()         | Numbered sequence    | 1, 2, 3... values    | for i in range(3)
dict.items()    | Key-value pairs      | key, value pairs     | for k, v in d.items()

WHAT IS A FUNCTION?
  A function is a reusable block of code that performs a specific task.
  
  Pattern:
    def function_name(parameters):
        # Code to execute
        return result

---

FUNCTION EXAMPLE 1: View Function (Django)
Location: profile_app/views.py (lines 11-36)

Code:
  @login_required(login_url='admin:login')
  def profile_view(request):
      """
      Display user profile with today's learning topics
      
      Demonstrates:
      - VARIABLES: user data storage, topics list, loop variables
      - LOOPS: Python for-loop to process topics
      """
      current_user = request.user
      # ... more code ...
      return render(request, 'profile.html', context)

Components:
  • @login_required: DECORATOR that protects the function (requires login)
  • def profile_view(request): Function definition with parameter 'request'
  • docstring: Triple-quoted documentation
  • Function body: Code that executes when function is called
  • return render(...): Returns rendered HTML template

Function call:
  Automatically called when user visits: http://localhost:8000/profile/

DECORATOR EXAMPLE: @login_required
Location: profile_app/views.py (line 11)

Code:
  @login_required(login_url='admin:login')
  def profile_view(request):
      # Function body

What it does:
  1. Before running profile_view():
     - Check if user is logged in
     If NO: Redirect to 'admin:login'
     If YES: Continue to function
  
  2. This decorator WRAPS the function to add behavior

Equivalent without decorator:
  def profile_view(request):
      if not request.user.is_authenticated:
          return redirect('admin:login')
      # Rest of function...
  
  # Decorator shortens this to just @login_required


4️⃣ PUTTING IT ALL TOGETHER
═══════════════════════════════════════════════════════════════════════════════

COMPLETE FLOW:

1. User visits: http://localhost:8000/profile/

2. Django routes to: profile_view(request) function

3. Function VARIABLES are created:
   current_user = request.user          # Get logged-in user
   today_topics = ["variables", ...]    # Create topics list
   user_data = {...}                    # Create user data dict

4. Function LOOP processes data:
   for index, topic in enumerate(today_topics):
       formatted_topic = f"Lesson {index}: {topic}"
       processed_topics.append(formatted_topic)

5. Function returns to template with CONTEXT (variables):
   context = {
       "profile": user_profile,
       "user_data": user_data,
       "today_topics": today_topics,
       "processed_topics": processed_topics,
   }
   return render(request, 'profile.html', context)

6. Template displays using VARIABLES:
   {{ user_data.username }}
   {{ user_data.email }}

7. Template LOOPS through topics:
   {% for topic in today_topics %}
       <li>{{ forloop.counter }}: {{ topic }}</li>
   {% endfor %}

8. HTML is rendered and sent to user's browser

9. User sees the beautiful profile page!
